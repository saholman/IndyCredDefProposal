Indy Credential Definition Specification Proposal

Currently only schemas consisting of a list of attributes are allowed to be used in Indy Credentials. This convention would allow any object that can be represented in JSON to be represented as an Indy Credential.

Advantages

  • A deterministic way to flatten any data model to be able to present it as an Indy Credential.

Rules and Quirks

  • If nothing appears in the second and third boxes, the system was unable to convert your JSON object
  • Currently arrays within arrays are not supported, although they will be supported in the official specification
  • Every instance of a schema must have every attribute, even if that attribute is blank.
